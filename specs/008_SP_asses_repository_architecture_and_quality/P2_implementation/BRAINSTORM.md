## The one deferral: D3 phase-2 monitoring wire-up

STATUS_REPORT §5 D3 decided to **keep and wire up** the monitoring layer (`ClassInfo`
consumed by the Logger, error codes surfaced in reports, `MLModelDescription`'s fate),
with the consumer arriving "alongside the DS/ML migration." That migration spec (009+)
**does not exist yet**, and in-repo there is still no consumer: `ClassInfo` is only
self-referenced in one transformer string, error codes are never branched on or reported,
`MLModelDescription` has zero consumers.

**Decision (grill 2026-07-16): defer the consumer-dependent D3 work to the DS/ML
migration spec.** Designing a monitoring consumer now — divorced from the real consumer —
is the exact speculative-infrastructure trap §4.3 warns about. P2 does only the *one*
consumer-independent piece: renaming the misleadingly-named `MetaClass` (it is an ABC, not
a metaclass). Spec 009 inherits the wire-up unchanged.
